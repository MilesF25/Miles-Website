BanjoDB

Final date TBD

B.A.N.J.O.

Backend Authenticated Note & Journal Organizer

I've been playing a lot of Banjo-Kazooie lately and decided to name this project after the duo. Thanks to my gaming buddy for providing the banjo image, You can check out the full repository here: BanjoDB on GitHub.

The Backstory

Recently, I was looking back at my senior project from college and felt a bit frustrated. At the time, I didn't feel I had the opportunity to truly build what I wanted, looking back, what I put together then was held together by hope and duct tape. I decided to treat this project as a remake, a chance to do it right using the skills I've picked up since graduating.

The original goal of my senior project was to create a secure database that was inaccessible to anyone without the correct password. I remember using BitLocker for data-at-rest protection and attempting to implement quantum-resistant algorithms. However, I didn't know enough about quantum computing at the time to implement it properly, i was trying to bite off more than i could chew and it would have required updating outdated Python libraries that were beyond my skill level then.

The Evolution

This new project is a simplified but much more robust version of that original idea. Since I'm currently studying for my Security+ certification, I wanted to implement the core pillars of modern security: Role-Based Access Control (RBAC), Secure File Handling, and Detailed Logging.

Technical Overview

  • The Interface: I used a Python frontend because, frankly, building a GUI in Rust is a special kind of pain. Python makes the user interaction seamless and easy.
  • The Secure Core: The entire backend is built in Rust. Python and Rust pass data back and forth, but nothing sensitive is ever held on the frontend. This simulates a real-world environment where the client is separated from the secure core.
  • Security Practices: I used Argon2 for password hashing, ensuring no credentials are ever stored in plain text.

Current Features

  • Users can create, edit, and save GVim-supported files directly into the database.
  • The system logs ownership and "Last Edited" timestamps for every entry.
  • The Admin account has total oversight, with the power to view, edit, or delete any file or account in the system.
  • Standard users are restricted to their own "vaults" and cannot access other users' data.

Looking Ahead

Overall, this was a rewarding way to spend my free time. It allowed me to get familiar with Rust/Python integration (via Maturin) and put database security practices into actual production. If I continue to iterate on this, I'd love to add a secure password recovery system, but for now, I'm proud of how far this "remake" has come.
