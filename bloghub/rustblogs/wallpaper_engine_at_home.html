Wallpaper testing

6/29/2026

This was just a small project i did to get more practice with Rust. I wanted to try working at a low level and get used to using tokio so just messing with the windows home wallpaper and music seemed like a good start. This program swaps through different wallpapers and plays music in the background This project wasnt that hard, the only tricky thing was figuring out how to use tokio. The issue I ran into was having one program run and the not the other. The goal using tokio was to have two functions run at the same time so I wasnt s the issue was becuase I was using thread::Sleep instead of tokio sleep on line 223 and how i orded the functions in main on line 42 (let _ = tokio::join!(papel_handle, radio_handle);). The issue with The original verion had papel_handle first then the radio handle. This caused the wallpaper to swap but then nothing else could happen, the music wouldn't start. It would be like the entire program froze, which is whats supposed to happen becuase I used thread::sleep. Thread::sleep basically pauses the entire program and i had it set for 50 seconds. To fix this I had to use tokio::sleep instead, this basically tells the wallpaper program to pause but still run the other functions in the meantime. The project is on my github here: https://github.com/MilesF25/Wallpaper-Engine-at-home/tree/main
